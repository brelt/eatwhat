## 1. 产品概述

### 1.1 产品定位
一句话描述：收集附近各大超市个性化菜品定制规划一周健康伙食食谱

### 1.2 目标市场
**澳大利亚** - 主要面向在澳华人家庭及对中式/亚洲饮食感兴趣的本地用户

### 1.3 目标用户
所有需要采购食材做饭的人/家庭，包括：
- 在澳华人家庭：希望用本地食材做出家乡味道
- 上班族：工作繁忙，希望高效规划每周饮食
- 留学生：预算有限，想吃得健康又省钱
- 年轻家庭：有孩子，注重营养均衡
- 健康饮食爱好者：注重营养均衡和饮食健康

### 1.4 核心功能概览
1. 抓取澳洲主流超市（Woolworths、Coles、ALDI、IGA等）菜品价格/打折情况
2. 抓取网上食谱数据（中式菜谱为主，兼顾西式及亚洲菜系）
3. 按照家庭人口、预算（AUD）、饮食习惯定制一周饮食计划
4. 按口味喜好定制食谱
5. 反馈机制，支持更换不满意的菜谱
6. 食谱分享给朋友

## 3. 核心页面功能需求

### 3.1 食谱规划页
**功能描述**：用户设置偏好后，系统生成一周7天21餐的食谱

#### 核心功能
- 显示一周食谱日历视图
- 支持按天查看详细食谱
- 支持替换单个食谱
- 显示总预算和营养信息
- 一键生成购物清单

#### 用户输入项
| 输入项 | 类型 | 说明 |
|--------|------|------|
| 家庭人数 | 选择器 | 1-6人 |
| 每周预算 | 滑块 | $50-200 AUD |
| 饮食偏好 | 多选 | 中餐、西餐、日韩、东南亚等 |
| 忌口设置 | 多选 | 素食、无麸质、低糖、过敏原等 |
| 烹饪难度 | 单选 | 简单/中等/高级 |
| 烹饪时长 | 单选 | <30分钟 / 30-60分钟 / >60分钟 |

### 3.2 菜谱详情页
**功能描述**：展示单个菜谱的详细信息

#### 内容模块
- 菜谱名称和图片
- 烹饪时长和难度
- 食材清单（含价格和超市信息）
- 烹饪步骤（图文）
- 营养信息
- 用户评价和评分
- 收藏/分享按钮

### 3.3 超市优惠页
**功能描述**：展示附近超市的打折商品

#### 功能模块
- 超市筛选（Woolworths/Coles/ALDI等）
- 商品分类筛选（蔬菜/肉类/海鲜等）
- 折扣排序（折扣力度/价格）
- 商品详情（图片/原价/折扣价/折扣比例）
- 添加到购物清单
- 查看相关食谱

### 3.4 购物清单页
**功能描述**：根据本周食谱自动生成购物清单

#### 功能模块
- 按超市分组显示
- 食材数量自动汇总
- 勾选已购买
- 预估总价格
- 导出/分享清单
- 采购路线优化建议

### 3.5 个人中心页
**功能描述**：用户设置和个人信息管理

#### 功能模块
- 个人信息编辑
- 饮食偏好设置
- 我的收藏食谱
- 历史食谱计划
- 账户设置
- 帮助与反馈

## 6. 数据需求

### 6.1 前端需要的数据接口

| 接口 | 描述 | 数据来源 | 优先级 |
|------|------|----------|--------|
| GET /api/recipes/recommended | 获取个性化推荐菜品 | 食谱库 + 用户偏好 | P0 |
| GET /api/supermarkets/nearby | 获取用户附近的超市列表 | 定位 + 超市数据库 | P0 |
| GET /api/deals | 获取打折商品信息 | 超市API/爬虫 | P0 |
| GET /api/user/preferences | 获取用户设置的偏好 | 用户数据库 | P0 |
| POST /api/meal-plan/generate | 生成一周食谱计划 | AI算法 + 食谱库 | P0 |
| PUT /api/meal-plan/:id/replace | 替换单个食谱 | 食谱库 | P1 |
| GET /api/shopping-list | 获取购物清单 | 食谱计划 | P0 |
| POST /api/recipes/:id/favorite | 收藏食谱 | 用户数据库 | P1 |
| GET /api/recipes/:id | 获取食谱详情 | 食谱库 | P0 |
| POST /api/recipes/:id/share | 分享食谱 | 分享服务 | P2 |

### 6.2 缓存策略
- 超市优惠数据：每日更新，本地缓存24小时
- 推荐食谱：根据用户行为实时调整
- 定位信息：本地缓存，可手动更新
- 用户偏好：本地缓存，同步到服务器

### 6.3 离线支持
- 已生成的食谱计划支持离线查看
- 购物清单支持离线访问
- 收藏的食谱支持离线查看

## 8. 技术实现建议

### 8.1 技术栈
- **框架**：React / Next.js / Vue.js
- **状态管理**：Redux / Zustand / Context API
- **样式**：Tailwind CSS / Styled Components
- **UI组件库**：Ant Design Mobile / Material-UI
- **图表库**：Chart.js / Recharts（营养信息展示）
- **地图服务**：Google Maps API（超市定位）

### 8.2 性能要求
- 首页加载时间 < 3秒（4G网络）
- 首屏渲染时间 < 1.5秒
- 页面切换动画流畅（60fps）
- 图片懒加载
- 代码分割（Code Splitting）

### 8.3 兼容性
- iOS 12+ / Android 8+
- 主流浏览器（Chrome、Safari、Firefox）
- 支持PWA（Progressive Web App）

### 8.4 安全性
- 用户定位信息加密存储
- HTTPS传输
- XSS防护
- CSRF防护
- 敏感数据不存储在LocalStorage
