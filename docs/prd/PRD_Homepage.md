## 1. 产品概述

### 1.1 产品定位
一句话描述：收集附近各大超市个性化菜品定制规划一周健康伙食食谱

### 1.2 目标市场
**澳大利亚** - 主要面向在澳华人家庭及对中式/亚洲饮食感兴趣的本地用户

### 1.3 目标用户
所有需要采购食材做饭的人/家庭，包括：
- 在澳华人家庭：希望用本地食材做出家乡味道
- 上班族：工作繁忙，希望高效规划每周饮食
- 留学生：预算有限，想吃得健康又省钱
- 年轻家庭：有孩子，注重营养均衡
- 健康饮食爱好者：注重营养均衡和饮食健康

### 1.4 核心功能概览
1. 抓取澳洲主流超市（Woolworths、Coles、ALDI、IGA等）菜品价格/打折情况
2. 抓取网上食谱数据（中式菜谱为主，兼顾西式及亚洲菜系）
3. 按照家庭人口、预算（AUD）、饮食习惯定制一周饮食计划
4. 按口味喜好定制食谱
5. 反馈机制，支持更换不满意的菜谱
6. 食谱分享给朋友

作为一个首次访问的用户，
我希望快速了解产品能为我做什么，
以便决定是否开始使用。
```

### 3.2 老用户回访
```
作为一个已设置偏好的老用户，

## 4. 交互设计要求

### 4.1 页面加载
- 首屏内容优先加载，确保3秒内可见
- 使用骨架屏（Skeleton）展示加载状态
- 图片懒加载，优化性能

### 4.2 响应式设计
- 支持移动端（主要）和桌面端
- 移动端优先设计原则
- 断点设置：375px / 768px / 1024px

### 4.3 交互反馈
- 按钮点击有明确反馈（颜色变化/动画）
- 列表支持下拉刷新
- 操作结果有Toast提示

## 5. 数据需求

### 5.1 首页需要的数据接口
| 接口 | 描述 | 数据来源 |
|------|------|----------|
| 推荐食谱 | 获取个性化推荐菜品 | 食谱库 + 用户偏好 |
| 附近超市 | 获取用户附近的超市列表 | 定位 + 超市数据库 |
| 超市优惠 | 获取打折商品信息 | 超市网站爬虫 |
| 用户偏好 | 获取用户设置的偏好 | 用户数据库 |

### 5.2 缓存策略
- 超市优惠数据：每日更新
- 推荐食谱：根据用户行为实时调整
- 定位信息：本地缓存，可手动更新

---

## 6. 非功能性需求

### 6.1 性能要求
- 首页加载时间 < 3秒（4G网络）
- 首屏渲染时间 < 1.5秒

### 6.2 兼容性
- iOS 12+ / Android 8+
- 主流浏览器（Chrome、Safari、Firefox）

### 6.3 安全性
- 用户定位信息加密存储
- HTTPS传输
